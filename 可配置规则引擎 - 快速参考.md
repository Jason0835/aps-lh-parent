# 可配置规则引擎 - 快速参考卡

## 🚀 快速开始

### 1. 执行初始化脚本
```bash
mysql -u user -p database < aps-lh/src/main/resources/sql/rule_engine_init.sql
```

### 2. 在代码中使用
```java
@Component
public class MyStrategy {
    @Resource
    private IScheduleRuleEngine ruleEngine;
    
    public void execute(LhScheduleContext context) {
        // 获取规则值
        int limit = ruleEngine.getDailyMouldChangeLimit(context.getFactoryCode());
        
        // 使用规则
        if (currentCount < limit) {
            // 业务逻辑
        }
    }
}
```

### 3. 修改规则值
```sql
-- 修改换模上限为 20
UPDATE T_LH_PARAMS 
SET PARAM_VALUE = '20' 
WHERE FACTORY_CODE = 'F001' AND PARAM_CODE = 'DAILY_MOULD_CHANGE_LIMIT';
```

## 🔧 常用 SQL

### 查询配置
```sql
-- 查询某工厂所有规则
SELECT * FROM T_LH_PARAMS 
WHERE FACTORY_CODE = 'F001' AND IS_DELETE = 0 
ORDER BY PARAM_CODE;

-- 查询特定规则
SELECT * FROM T_LH_PARAMS 
WHERE PARAM_CODE = 'DAILY_MOULD_CHANGE_LIMIT';
```

### 修改配置
```sql
-- 修改单个规则
UPDATE T_LH_PARAMS 
SET PARAM_VALUE = '20' 
WHERE FACTORY_CODE = 'F001' AND PARAM_CODE = 'DAILY_MOULD_CHANGE_LIMIT';

-- 批量修改 (多个工厂)
UPDATE T_LH_PARAMS 
SET PARAM_VALUE = '25', REMARK = '提升产能' 
WHERE PARAM_CODE = 'DAILY_MOULD_CHANGE_LIMIT';
```

### 新增配置
```sql
INSERT INTO T_LH_PARAMS (FACTORY_CODE, PARAM_CODE, PARAM_VALUE, PARAM_NAME, REMARK) 
VALUES ('F002', 'DAILY_MOULD_CHANGE_LIMIT', '12', '每日换模上限', 'F002 专用配置');
```

### 删除配置
```sql
-- 逻辑删除
UPDATE T_LH_PARAMS 
SET IS_DELETE = 1 
WHERE FACTORY_CODE = 'F001' AND PARAM_CODE = 'XXX';
```

## 💡 最佳实践

### ✅ 推荐做法
```java
// 提前缓存规则值
int limit = ruleEngine.getDailyMouldChangeLimit(factoryCode);
for (SkuScheduleDTO sku : skus) {
    if (count < limit) {
        // 业务逻辑
    }
}

// 记录规则使用情况
log.info("使用换模上限规则：factory={}, limit={}", factoryCode, limit);
```

### ❌ 不推荐做法
```java
// 循环内重复调用
for (SkuScheduleDTO sku : skus) {
    if (count < ruleEngine.getDailyMouldChangeLimit(factoryCode)) {
        // 性能较差
    }
}

// 忽略返回值
int limit = ruleEngine.getDailyMouldChangeLimit(factoryCode);
// ... 很久之后才使用
```

## ⚡ 故障排查

### 规则未生效？
1. 检查数据库记录是否存在
2. 确认 IS_DELETE = 0
3. 等待 5 分钟或手动清缓存
4. 查看日志是否有异常

### 手动清缓存
```java
@Resource
private DatabaseScheduleRuleEngine ruleEngine;

// 清除单个工厂
ruleEngine.clearCache("F001");

// 清除所有
ruleEngine.clearCache(null);
```

### 查看日志
```bash
# 查找规则相关日志
tail -f app.log | grep "规则引擎"
tail -f app.log | grep "DatabaseScheduleRuleEngine"
```

## 🎯 核心 API

### 获取整数规则
```java
int value = ruleEngine.getDailyMouldChangeLimit(factoryCode);
```

### 获取小数规则
```java
double value = ruleEngine.getCapsulePreheatHours(factoryCode);
```

### 获取效率因子
```java
double efficiency = ruleEngine.getShiftEfficiencyFactor(factoryCode, "MORNING");
```

### 清除缓存
```java
ruleEngine.clearCache(factoryCode); // factoryCode=null 清除所有

```
