# 续作多机台分配计划量需求描述（最终版）



## 1. 需求背景

当前硫化排程中，存在同一个 SKU 在多台机台上续作生产的场景。例如 MES 在机信息中：

```text
SKU1 - 机台1
SKU1 - 机台2
```

当后续日计划量下降时，如果仍然让所有在机机台继续生产，可能会造成不必要的占机、超排或后续换模集中。因此需要支持：

```text
续作 SKU 多机台分配计划量
续作 SKU 降模排产
续作 SKU 多机台收尾错峰
```

本需求是对已有“新增排产 SKU 多机台动态分配计划量”逻辑的补充，但两者适用范围不同：

| 类型           | 说明                                                         |
| -------------- | ------------------------------------------------------------ |
| 新增排产多机台 | 当前 SKU 新进入排程，单机台产能不足时，判断是否增加机台      |
| 续作多机台     | 当前 SKU 已在多台机台上生产，后续计划量下降时，判断是否减少机台 |

---

## 2. 适用范围

本需求仅适用于：

```text
续作排产 + 同 SKU 多机台在机
```

包括但不限于：

1. MES 在机快照中同一个 SKU 同时存在于多台机台；
2. 上一轮排程结果中同一个 SKU 已经在多台机台续作；
3. 当前排程需要延续该 SKU 的生产；
4. 后续日计划量下降，需要判断是否减少续作机台。

不适用于：

1. 新增排产 SKU 的初次选机台；
2. 新增排产 SKU 的新增机台判断；
3. 非续作 SKU 的换模新增排产；
4. 非同 SKU 多机台续作场景。

---

## 3. 基础数据定义

### 3.1 业务参数

| 参数           | 含义                                             |
| -------------- | ------------------------------------------------ |
| 班产           | 单台机台单班产量                                 |
| dayN           | 月计划中某生产日的计划量                         |
| 余量           | 月计划剩余需排量                                 |
| 胎胚库存       | 当前 SKU 对应胎胚库存                            |
| SKU 类型       | 正规 / 量试 / 试制                               |
| 是否收尾       | 当前 SKU 是否为收尾场景                          |
| 胶囊已使用次数 | 当前机台胶囊使用次数，用于降模时判断优先下机机台 |
| 机台编码       | 胶囊使用次数相同时，用于下机排序                 |

### 3.2 本案例基础数据

```text
班产 = 16
余量 = 800
胎胚库存 = 0
SKU 类型 = 正规且非收尾
月计划：day1 = 64，day2 = 48，day3 = 48
前一次 MES 在机信息：SKU1 在机台1、机台2 同时生产
```

机台胶囊使用情况：

| 机台  | 胶囊已使用次数 |
| ----- | -------------: |
| 机台1 |             10 |
| 机台2 |              5 |

## 4. 续作降模排产规则

### 4.1 什么是降模排产

降模排产指：

```text
同一个续作 SKU 原来在多台机台上生产，
但后续日计划量下降，
不再需要这么多机台同时生产，
因此保留部分机台继续生产，
其余机台下机释放。
```

也可以理解为：

```text
新增排产是“加机台”；
续作降模是“减机台”。
```

### 4.2 触发前提

当满足以下条件时，需要判断是否降模：

1. 当前 SKU 是续作 SKU；
2. 同一个 SKU 当前在多台机台上续作；
3. 当前或后续日计划量相比前一天出现下降；
4. 当前在机多台机台的最大日产能大于当日需保障量；
5. 部分机台已经足够支撑当日需保障量。

### 4.3 当日需保障量

非收尾续作场景下：

```text
当日需保障量 = 当日月计划 dayN
```

如果项目已有欠产滚动逻辑，可扩展为：

```text
当日需保障量 = 当日月计划 dayN + 前序累计欠产
```

本案例没有前序欠产，因此按 dayN 判断。

### 4.4 当前在机最大日产能

```text
当前在机最大日产能 = 当前续作机台数 × 当日每台机台可生产班次数 × 班产
```

例如：

```text
班产 = 16
每天 3 个班次
单台日产能 = 16 × 3 = 48
两台日产能 = 48 × 2 = 96
```

### 4.5 是否需要降模

如果：

```text
当前在机最大日产能 > 当日需保障量
```

且：

```text
部分机台的累计可用产能 >= 当日需保障量
```

则说明当前续作机台数量过多，可以降模。

## 5. 降模后保留机台数量规则

### 5.1 单台产能相同

如果每台机台当天可生产班次数相同、班产相同，则：

```text
需要保留机台数 = CEIL(当日需保障量 / 单台当日可排产能)
```

例如：

```text
day2 = 48
单台当日产能 = 16 × 3 = 48
需要保留机台数 = CEIL(48 / 48) = 1
```

因此只需要保留 1 台机台。

### 5.2 单台产能不同

如果不同机台当天可生产班次数不同，例如某些机台存在换模、停机、保养、异常，则不能简单用平均产能计算。

应按保留优先级排序机台后逐台累计：

```text
按保留优先级排序机台；
逐台累计可用产能；
直到累计可用产能 >= 当日需保障量。
```

---

## 6. 降模时下机优先级

当同 SKU 多机台续作需要降模时，需要决定哪些机台保留，哪些机台下机。

### 6.1 下机优先级

```text
胶囊已使用次数少的机台优先下机；
如果胶囊已使用次数相同，机台编码大的优先下机。
```

排序规则：

```text
capsuleUsedCount ASC
machineCode DESC
```

### 6.2 保留优先级

保留机台排序与下机排序相反：

```text
胶囊已使用次数多的机台优先保留；
如果胶囊已使用次数相同，机台编码小的优先保留。
```

排序规则：

```text
capsuleUsedCount DESC
machineCode ASC
```

## 7. 续作非收尾排计划量规则

### 7.1 正规 / 量试非收尾

续作 SKU 如果是：

```text
正规非收尾
或
量试非收尾
```

则规则为：

```text
先判断需要保留几台机台；
再对保留下来的已开机台当天剩余可用班次补满班产。
```

也就是说，一旦确定某台机台当天继续生产这个 SKU，该机台当天后续可生产班次要按班产排满。

例如：

```text
班产 = 16
某台保留机台当天可生产 C1、C2、C3
```

则该机台排产：

```text
C1 = 16
C2 = 16
C3 = 16
```

注意：补满班产只作用于**保留下来的续作机台**。

不能理解为：

```text
所有原在机机台都必须补满。
```

正确顺序是：

```text
先判断是否降模以及保留哪些机台；
再对保留机台执行当天剩余班次补满。
```

### 7.2 试制非收尾

续作 SKU 如果是试制非收尾，则：

```text
严格按照月计划 dayN 排产；
不允许为了补满班次而超排。
```

例如：

```text
dayN = 40
班产 = 16
```

可排：

```text
C1 = 16
C2 = 16
C3 = 8
```

不能排成：

```text
C1 = 16
C2 = 16
C3 = 16
```

---

## 8. 续作收尾排计划量规则

如果续作 SKU 是收尾场景，不管 SKU 类型是：

```text
正规
量试
试制
```

都必须严格按照月计划 dayN 或当日收尾目标量排产。

规则：

```text
收尾场景不补满班次；
不允许超排；
剩余多少排多少；
达到目标后停止。
```

例如：

```text
班产 = 16
剩余目标 = 14
```

只能排：

```text
14
```

不能补满到：

```text
16
```

## 9. 续作同 SKU 多机台收尾换模错开规则

### 9.1 触发前提

续作排产中，如果出现：

```text
同 SKU 多机台收尾
```

则需要结合后续换模错峰判断是否需要错开收尾。

注意：

```text
强调的是“同 SKU 多机台收尾时，再结合换模错峰去判断是否需要错开”，
不是“同胎胚一上来就禁止同班次换模”。
```

也就是说，不能一开始发现同胎胚/共用胎胚就直接禁止同班次换模。

正确触发顺序：

```text
先判断是否同 SKU 多机台收尾；
再判断这些机台后续是否存在换模错峰需求；
最后再决定是否调整尾量分配。
```

### 9.2 复用新增排产同 SKU 多机台收尾规则

续作排产的同 SKU 多机台收尾错峰规则，与新增排产中的同 SKU 多机台收尾规则保持一致。

#### 收尾班次是晚班

```text
不强制错开收尾；
只校验换模均衡。
```

原因：

```text
晚班不能换模。
```

晚班收尾后，新 SKU 只能等后续早班换模、中班开产。此时强行把尾量挪到早班或中班，反而可能浪费有效产能。

#### 收尾班次是早班或中班

```text
需要尝试错开收尾；
同时校验换模均衡。
```

错开的目标不是形式上的“收尾班次不同”，而是：

```text
尽量不浪费产能；
同时保证换模均衡。
```

### 9.3 换模均衡约束

续作收尾换模也需要校验换模均衡。

当前规则：

```text
早班换模次数 <= 8
中班换模次数 <= 7
一天换模总次数 <= 15
```

如果项目中已有参数，则复用参数。

如果没有，建议作为硫化参数配置，不要写死。

## 10. 案例推演

### 10.1 案例基础数据

```text
班产 = 16
余量 = 800
胎胚库存 = 0
SKU 类型 = 正规且非收尾
月计划：day1 = 64，day2 = 48，day3 = 48
前一次 MES 在机信息：SKU1 机台1，SKU1 机台2
```

胶囊使用次数：

```text
机台1 胶囊已使用次数 = 10
机台2 胶囊已使用次数 = 5
```

班次说明：

```text
第一天：2 个班次
第二天：3 个班次
第三天：3 个班次

第一天单台日产能 = 16 × 2 = 32
第一天两台日产能 = 32 × 2 = 64

第二天/第三天单台日产能 = 16 × 3 = 48
第二天/第三天两台日产能 = 48 × 2 = 96
```

### 10.2 第一天：day1 = 64

当前两台机台都在机。

第一天只有 2 个班次。

两台最大日产能：

```text
2 台 × 2 个班次 × 16 = 64
```

刚好满足 day1。

因此 day1 不需要降模，两台都继续生产。

| 机台  |   C1 |   C2 | 胶囊已使用次数 | day1 合计 |
| ----- | ---: | ---: | -------------: | --------: |
| 机台1 |   16 |   16 |             10 |        32 |
| 机台2 |   16 |   16 |              5 |        32 |
| 合计  |   32 |   32 |              - |        64 |

### 10.3 第二天：day2 = 48

如果两台机台都继续生产，则最大日产能：

```text
2 × 3 × 16 = 96
```

但 day2 只需要：

```text
48
```

判断：

```text
当前在机最大日产能 96 > day2 目标量 48
```

单台日产能：

```text
3 × 16 = 48
```

判断：

```text
单台日产能 48 >= day2 目标量 48
```

结论：

```text
一台机台足够支撑 day2 的月计划量；
因此需要降模，下掉一台机台。
```

下机规则：

```text
胶囊已使用次数少的机台优先下机。
```

由于：

```text
机台2 胶囊已使用次数 5 < 机台1 胶囊已使用次数 10
```

所以：

```text
第二天优先下机：机台2
第二天保留生产：机台1
```

第二天结果：

| 机台  |   C1 |   C2 |   C3 | day2 合计 | 状态     |
| ----- | ---: | ---: | ---: | --------: | -------- |
| 机台1 |   16 |   16 |   16 |        48 | 继续续作 |
| 机台2 |    0 |    0 |    0 |         0 | 降模下机 |
| 合计  |   16 |   16 |   16 |        48 | -        |

### 10.4 第三天：day3 = 48

当前保留续作机台：

```text
机台1
```

机台1 当天最大产能：

```text
3 × 16 = 48
```

判断：

```text
机台1 当天产能 48 >= day3 目标量 48
```

因此第三天仍由机台1继续生产。

| 机台  |   C1 |   C2 |   C3 | day3 合计 |
| ----- | ---: | ---: | ---: | --------: |
| 机台1 |   16 |   16 |   16 |        48 |
| 机台2 |    0 |    0 |    0 |         0 |
| 合计  |   16 |   16 |   16 |        48 |

## 11. 最终排产结果

| 日期 | 机台1 | 机台2 | 合计 | 说明                              |
| ---- | ----: | ----: | ---: | --------------------------------- |
| day1 |    32 |    32 |   64 | 第一天 2 个班次，两台续作，不降模 |
| day2 |    48 |     0 |   48 | 日计划下降，降模，机台2 下机      |
| day3 |    48 |     0 |   48 | 机台1 继续续作                    |

最终结果符合：

1. 续作多机台日计划下降时进行降模；
2. 胶囊已使用次数少的机台优先下机；
3. 胶囊已使用次数相同时，机台编码大的优先下机；
4. 正规非收尾场景，保留机台当天剩余班次补满班产；
5. 非收尾场景不进入同 SKU 多机台收尾错峰逻辑；
6. 续作逻辑与新增排产逻辑分开处理。

---

## 12. 建议实现要点

1. 续作多机台逻辑需要独立于新增排产多机台逻辑；
2. 先识别同 SKU 多机台续作集合；
3. 按 dayN 逐日判断是否需要降模；
4. 根据当日需保障量和机台可用产能计算保留机台数量；
5. 按“胶囊已使用次数少优先下机，机台编码大优先下机”确定下机机台；
6. 正规/量试非收尾：保留机台补满当天可生产班次；
7. 试制非收尾：严格按 dayN，不补满；
8. 收尾：不管 SKU 类型，严格按目标量，不补满，不超排；
9. 同 SKU 多机台收尾时，再结合换模错峰判断是否需要错开；
10. 换模均衡参数需要可配置，不要写死。
