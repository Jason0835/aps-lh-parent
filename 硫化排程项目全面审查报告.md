### 1.6 异常处理分析

#### 当前问题

1. **异常处理不一致**
   - 有些地方直接抛出`IllegalArgumentException`（ScheduleStrategyFactory:140）
   - 有些地方调用`context.interruptSchedule()`
   - 有些地方静默处理exception（MesNotifyListener:28）

2. **异常转换不规范**
   - 大多数捕获的Exception没有转换为ScheduleException
   - 没有建立一致的异常映射规则

3. **异常日志记录不完整**
   - 某些异常处理后只记录日志，没有记录完整的堆栈跟踪

#### 改进建议

```java
// 统一异常处理策略
public class ScheduleExceptionHandler {
    public void handle(Exception e, LhScheduleContext context) {
        if (e instanceof ScheduleException) {
            // 业务异常：记录并中断
            context.interruptSchedule(e.getMessage());
        } else {
            // 系统异常：转换并记录堆栈
            ScheduleException se = new ScheduleException(
                context.getCurrentStep(), 
                ScheduleErrorCode.SYSTEM_ERROR, 
                e.getMessage(), e);
            context.interruptSchedule(se.getMessage());
        }
    }
}
```

## 三、排程算法层面

### 3.1 算法设计概述

**整体策略**：贪心算法 + 规则优先级

```
SKU优先级排序 → 机台匹配 → 约束校验 → 班次分配
```

### 3.2 SKU优先级排序（DefaultSkuPriorityStrategy）

**排序规则（优先级从高到低）**：

```java
Comparator<SkuScheduleDTO> comparator = Comparator
    // 1. 有发货要求的优先
    .comparingInt((SkuScheduleDTO s) -> s.isDeliveryLocked() ? 0 : 1)
    // 2. 延误天数越多越优先（降序）
    .thenComparingInt((SkuScheduleDTO s) -> -s.getDelayDays())
    // 3. 收尾SKU优先；收尾日越晚的越先上机
    .thenComparingInt((SkuScheduleDTO s) -> isEnding(s) ? 0 : 1)
    .thenComparingInt((SkuScheduleDTO s) -> -s.getEndingDaysRemaining())
    // 4. 供应链优先级：高优先级(04) > 周期排产(05) > 中优先级(06) > 搭配排产(07)
    .thenComparingInt(this::getSupplyChainPriorityOrder);
```

**时间复杂度**：O(n log n)

**评价**：规则设计合理，但固定排序无法适应复杂场景

### 3.3 机台匹配算法（DefaultMachineMatchStrategy）

#### 匹配流程

```
1. 获取定点机台限制集合
2. 获取SKU的模具号列表
3. 获取已被占用的模具集合（共用模保护）
4. 过滤候选机台
   ├── 机台状态=启用
   ├── 寸口范围匹配
   ├── 模数不超最大模台数
   └── 模具未被占用
5. 多维度排序候选机台
```

#### 候选机台排序规则

```java
candidates.sort(
    // 1. 同规格优先
    Comparator.comparingInt(m -> sku.getSpecCode().equals(m.getPreviousSpecCode()) ? 0 : 1)
    // 2. 收尾时间升序（±20分钟容差视为相同）
    .thenComparing((m1, m2) -> compareEndingTime(m1, m2, toleranceMinutes))
    // 3. 相同英寸优先
    .thenComparingInt(m -> isSameInch(skuInch, m.getPreviousProSize()) ? 0 : 1)
    // 4. 英寸差距最小的优先
    .thenComparingDouble(m -> calcInchDistance(skuInch, m.getPreviousProSize()))
    // 5. 胶囊使用次数少的优先
    .thenComparingInt(MachineScheduleDTO::getCapsuleUsageCount)
);
```

**时间复杂度**：O(n × m log m)，其中n为SKU数量，m为候选机台数

### 3.4 产能计算策略（DefaultCapacityCalculateStrategy）

#### 计算公式

| 计算项 | 公式 | 说明 |
|--------|------|------|
| 班产能 | `(班次时间秒数 / 硫化时间秒数) × 模数` | 班次时间=28800秒（8小时） |
| 日产能 | `(24 × 3600 / 硫化时间秒数) × 模数` | 24小时连续生产 |
| 首班产量 | `(首班可用时间秒数 / 硫化时间秒数) × 模数` | 考虑开产时间偏移 |
| 开产时间 | `MAX(基础开产时间, 保养后时间, 维修后时间)` | 取三者最大值 |

#### 开产时间计算

```
基础开产时间 = 前SKU收尾时间 + 换模总耗时（8小时）
保养后开产时间 = 保养开始(8:00) + 保养时长(7h) + 换模(8h)
维修后开产时间 = 维修开始(8:00) + 维修时长 + 换模(8h)
最终开产时间 = MAX(基础, 保养后, 维修后)
```

### 3.5 换模均衡策略（DefaultMouldChangeBalanceStrategy）

#### 约束条件

| 约束 | 值 | 说明 |
|------|-----|------|
| 每日换模上限 | 15台 | 硬编码在常量类 |
| 早班换模上限 | 8台 | 06:00-14:00 |
| 中班换模上限 | 7台 | 14:00-22:00 |
| 夜班换模上限 | 0台 | 22:00-06:00禁止换模 |

#### 分配算法

```
1. 检查是否在禁止换模时段（20:00-次日06:00）
   └── 是 → 延后到次日早班
2. 判断当前时间所属班次
   ├── 早班 → 检查早班余量，满则移到中班
   ├── 中班 → 检查中班余量，满则延后到次日早班
   └── 夜班 → 直接延后到次日早班
3. 最多向后探索5天
```

                                  String factoryCode) {
        double score = 0;
        
        // 规格匹配得分
        double specWeight = ruleEngine.getWeight(factoryCode, "SPEC_MATCH_WEIGHT");
        score += specWeight * calcSpecMatchScore(machine, sku);

        double timeWeight = ruleEngine.getWeight(factoryCode, "TIME_DISTANCE_WEIGHT");
        score += timeWeight * calcTimeScore(machine);
        
        // 英寸距离得分
        double inchWeight = ruleEngine.getWeight(factoryCode, "INCH_DISTANCE_WEIGHT");
        score += inchWeight * calcInchScore(machine, sku);
        
        // 胶囊使用得分
        double capsuleWeight = ruleEngine.getWeight(factoryCode, "CAPSULE_USAGE_WEIGHT");
        score += capsuleWeight * calcCapsuleScore(machine);
        
        return score;
    }

public class LocalSearchOptimizer {
    
    public void optimize(LhScheduleContext context) {
        List<LhScheduleResult> results = context.getScheduleResultList();
        boolean improved = true;
        int maxIterations = 100;
        int iteration = 0;
        
        while (improved && iteration < maxIterations) {
            improved = false;
            // 2-opt交换：尝试交换两个SKU的机台分配
            for (int i = 0; i < results.size() - 1; i++) {
                for (int j = i + 1; j < results.size(); j++) {
                    if (canSwap(results.get(i), results.get(j), context)) {
                        double currentScore = evaluateScore(results, context);
                        swap(results.get(i), results.get(j));
                        double newScore = evaluateScore(results, context);
                        if (newScore > currentScore) {
                            improved = true;
                        } else {
                            // 回滚
                            swap(results.get(i), results.get(j));
                        }
                    }
                }
            }
            iteration++;
        }
    }
}
