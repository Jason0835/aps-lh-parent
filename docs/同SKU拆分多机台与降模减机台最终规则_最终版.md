## 2. 适用范围

### 2.1 加机台 / 拆分多机台适用范围

加机台 / 拆分多机台的最终机台分配动作适用于：

1. 新增排产，也就是换模场景。

换活字块排产场景需要单独区分：

1. 换活字块阶段先使用当前可衔接的换活字块机台进行排产；
2. 如果该单台机台产能无法满足窗口 dayN 日计划量和欠产追补要求，剩余量不应继续在换活字块阶段强行新增换活字块机台；
3. 剩余量应继续进入新增排产 / 换模阶段，作为待排需求通过换模分配到其他可用机台；
4. 新增排产 / 换模阶段再使用本文档的 dayN 日计划 + shortageLookAheadDays 欠产追补 + 最小增量机台数模拟逻辑判断是否需要多机台、哪一天加机台、加几台。

注意：本文档不展开换活字块匹配条件。换活字块具体匹配逻辑以项目现有代码和现有业务规则为准。

### 2.2 降模减机台适用范围

降模减机台只适用于：

1. 续作排产场景。

续作排产场景指：

1. 前一次 MES 在机信息中，同一个 SKU 已经在一台或多台机台上生产；
2. 当前排程窗口继续生产该 SKU。

### 2.3 边界要求

1. 新增排产 / 换模场景：执行加机台 / 拆分多机台，不做降模减机台。
2. 换活字块场景：先执行当前可衔接机台的换活字块承接；若单台产能不足，剩余量回流到新增排产 / 换模阶段，不在换活字块阶段直接强行新增其他换活字块机台。
3. 续作排产场景：只做降模减机台判断，不套用新增排产 / 换活字块剩余量回流后的加机台逻辑。
4. 不要把降模减机台逻辑应用到新增排产或换活字块场景。
5. 不要把新增排产 / 换模的多机台加机台动作误写成换活字块阶段内部直接加机台。

## 3. 核心目标

同一个 SKU 在当前 3 天排程窗口、8 个班次内，如果单台机台无法满足排产需求，可以拆分到多台机台生产。

但多机台拆分不能简单按以下条件直接从窗口第一天一次性加机台：

```text
单机台窗口有效产能 < SKU 窗口目标量
```

正确逻辑是：

1. 用窗口目标量判断该 SKU 是否可能需要多机台；
2. 用 dayN 日计划量和欠产追补能力判断哪一天需要加机台；
3. 用最小增量原则判断当天需要新增几台机台；
4. 用 activeMachines 维护当前已经启用的机台集合；
5. 每新增一台机台，都重新模拟当前日及后续追补范围内的产能；
6. 最终得到：
   - day1 新增几台；
   - day2 新增几台；
   - day3 新增几台；
   - 当前 SKU 最终启用几台机台。

当前排程窗口为：

```text
3 天、8 个班次
```

需要明确区分：

### 4.1 当前排程窗口

当前排程窗口用于实际排产、目标量分配、欠产追补、加机台判断。

```text
当前排程窗口 = 3 天、8 个班次
```

### 4.2 结构五天内收尾

结构五天内收尾只用于 SKU 排序层级判断。

2. 结构五天内收尾不能直接作为是否拆分多机台的判断依据；
3. 不要把结构五天内收尾和当前 3 天排程窗口收尾混用。

## 5. 目标量计算

### 5.1 正规 / 量试 / 小批量非收尾

窗口目标量表达为：

```text
窗口目标量 = day1 + day2 + day3 + 历史允许追补欠产
```

其中 day1、day2、day3 来自当前排程窗口对应的月计划 dayN。

### 5.2 历史允许追补欠产处理口径

窗口目标量可以表达为：

```text
窗口目标量 = day1 + day2 + day3 + 历史允许追补欠产
```

但在逐日模拟中，历史允许追补欠产必须作为 day1 的初始 carryShortage，或等价并入 day1 需求，避免窗口目标量和逐日模拟口径分叉。

推荐实现方式：

```text
initialCarryShortage = 历史允许追补欠产

day1.todayRequired = initialCarryShortage + day1Plan
day2.todayRequired = carryShortage + day2Plan
day3.todayRequired = carryShortage + day3Plan
```

注意：

1. 历史允许追补欠产只能进入一次；
2. 不能既作为 day1 初始 carryShortage，又在后续累计需求中重复追加；
3. 总量判断与逐日模拟必须使用同一口径。

### 5.3 试制 SKU

目标量为：

```text
目标量 = 试制计划量
```

要求：

1. 严格目标量；
2. 不允许超排；
3. 只能使用单控机台；
4. 即使拆分多台单控机台，也只能排到目标量为止；
5. 如果单控机台不足，输出未排原因。

### 5.4 收尾 SKU

收尾目标量为：

```text
收尾目标量 = MAX(余量, 胎胚库存)
```

要求：

1. 收尾判断针对 SKU 整体，不是针对单台机台；
2. 严格目标量；
3. 不允许超排；
4. 如果单台机台能在当前排程窗口内完成收尾，则不拆多机台；
5. 如果单台机台无法完成，且多台机台可以完成，则按多机台逻辑判断加机台。

## 6. 欠产追补参数 shortageLookAheadDays

新增参数：

```text
shortageLookAheadDays
```

含义：

```text
当前日欠产后，允许向后模拟追补的天数。
```

默认值：

```text
2
```

当前项目排程窗口为 3 天，因此最大实际有效值不能超过窗口最后一天 day3。

### 6.1 配置为 1

```text
shortageLookAheadDays = 1
```

含义：

1. day1 欠产时，只判断 day2 是否可以追回；
2. day2 欠产时，只判断 day3 是否可以追回；
3. day3 欠产时，不能继续往后追补。

### 6.2 配置为 2

```text
shortageLookAheadDays = 2
```

含义：

1. day1 欠产时，可以判断 day2 + day3 是否可以追回；
2. day2 欠产时，最多只能判断 day3；
3. day3 欠产时，不能继续往后追补。

### 6.3 追补结束日

无论 shortageLookAheadDays 配置多大，都不能超过当前 3 天排程窗口。

实际追补结束日：

```text
lookAheadEndDay = min(当前日 + shortageLookAheadDays, 窗口最后一天 day3)
```

## 7. 拆分多机台判断逻辑

请采用以下方式：

```text
dayN 日计划 + 欠产追补 + 最小增量机台数模拟
```

不要只按窗口总量粗暴计算：

```text
目标量 / 单台窗口产能
```

原因是不同机台可能存在：

1. 有效班次不同；
2. 班产不同；
3. 换模损失不同；
4. 已占用班次不同；
5. 停产、日历、不可排班次不同；
6. 单控机台、普通机台、特殊机台约束不同；
7. day1、day2、day3 每天可用产能不同。

### 7.1 换活字块剩余量回流新增排产规则

换活字块排产的 SKU，如果当前可衔接的单台机台有效产能无法满足窗口 dayN 日计划量和欠产追补要求，剩余量需要继续进入新增排产阶段。

核心规则：

1. 换活字块阶段只负责当前可衔接机台的承接排产；
2. 当前换活字块机台可排多少，先按有效产能和目标量排多少；
3. 当前换活字块机台无法承接的剩余量，形成 `remainingQtyForNewSchedule`；
4. `remainingQtyForNewSchedule` 继续进入新增排产 / 换模阶段；
5. 新增排产 / 换模阶段根据 SKU 与机台匹配、机台优先级、dayN 日计划、shortageLookAheadDays 欠产追补和最小增量原则，将剩余量通过换模分配到其他机台；
6. 不能在换活字块阶段为了满足剩余量而强行把其他机台也按换活字块方式加入；
7. 这样可以保持换活字块承接逻辑与新增排产换模逻辑边界清晰。

示例：

```text
SKU A 窗口需求 = 160
换活字块可衔接机台 M1 有效产能 = 96

M1 在换活字块阶段排 96
剩余量 = 160 - 96 = 64

剩余 64 进入新增排产 / 换模阶段，
由新增排产逻辑选择其他可用机台通过换模承接。
```

注意：

1. 剩余量进入新增排产阶段后，仍需遵守 SKU 类型约束、收尾/试制严格目标量约束、单控/普通/特殊机台约束；
2. 剩余量不能重复计入目标量；
3. 换活字块已排产量和新增排产剩余量合计不能超过 SKU 当前场景允许排产目标量；
4. 正规 / 量试 / 小批量非收尾在新增排产阶段仍可按已启用机台补满有效班次，但不能为了单纯补满空闲产能无理由新增机台。

## 8. activeMachines 规则

维护 activeMachines：

```text
activeMachines = 当前已经启用并可继续生产该 SKU 的机台集合
```

### 8.1 新增排产 / 换模场景

1. activeMachines 初始为当前选中的第一台机台；
2. 如果当前机台组合不满足需求，再按机台优先级逐台新增候选机台。

### 8.2 换活字块场景

1. activeMachines 初始为当前换活字块可衔接的单台机台；
2. 换活字块阶段只计算该可衔接机台在窗口内的有效可排产能和可承接数量；
3. 如果该机台产能可以满足窗口 dayN 日计划量和欠产追补要求，则不产生新增排产剩余量；
4. 如果该机台产能无法满足窗口 dayN 日计划量和欠产追补要求，差额形成 remainingQtyForNewSchedule；
5. remainingQtyForNewSchedule 进入新增排产 / 换模阶段，再由新增排产多机台逻辑分配到其他机台；
6. 不在换活字块阶段直接按换活字块方式逐台新增其他机台；
7. 换活字块具体匹配逻辑以项目现有代码和现有业务规则为准，本文档不展开其匹配条件。

### 8.3 累积规则

1. day1 新增的机台，day2 / day3 继续属于 activeMachines；
2. 每天新增机台数量是增量，不是每天独立重新计算；
3. 不允许 day1 加过的机台在 day2 计算时被忘掉。

## 9. 逐日判断流程

按排程窗口日期顺序处理：

```text
day1 -> day2 -> day3
```

每天计算：

```text
todayRequired = carryShortage + 当前 dayN 日计划量

todayCapacity = activeMachines 当前日有效可排产能
```

其中 carryShortage 表示前序未完成的欠产。day1 的 carryShortage 初始值应等于历史允许追补欠产，或者等价并入 day1 需求。

### 9.1 当天产能满足需求

如果：

```text
todayCapacity >= todayRequired
```

表示当前已启用机台可以满足当天需求。

处理：

1. 当前日不新增机台；
2. carryShortage = 0。

### 9.2 当天产能不足

如果：

```text
todayCapacity < todayRequired
```

表示当前日产生欠产。

计算：

```text
dayShortage = todayRequired - todayCapacity
```

根据 shortageLookAheadDays 计算允许追补的结束日：

```text
lookAheadEndDay = min(当前日 + shortageLookAheadDays, day3)
```

计算当前日到 lookAheadEndDay 的累计需求：

```text
cumulativeRequired =
    carryShortage
    + 当前日 dayN 计划量
    + 当前日之后直到 lookAheadEndDay 的各日 dayN 计划量
```

计算当前日到 lookAheadEndDay 的累计产能：

```text
cumulativeCapacity =
    activeMachines 在当前日到 lookAheadEndDay 的有效可排产能合计
```

### 9.3 允许追补范围内可以追回

如果：

```text
cumulativeCapacity >= cumulativeRequired
```

表示在允许追补范围内可以追回欠产。

处理：

1. 当前日不新增机台；
2. carryShortage = dayShortage。

### 9.4 允许追补范围内追不回

如果：

```text
cumulativeCapacity < cumulativeRequired
```

表示在允许追补范围内追不回欠产。

处理：

1. 当前日需要新增机台；
2. 从候选机台列表中按机台优先级逐台加入 activeMachines；
3. 每加入一台机台，都重新计算 cumulativeCapacity；
4. 直到 cumulativeCapacity >= cumulativeRequired，或者没有候选机台为止；
5. 本轮加入的机台数量，就是当前日新增机台数量。

### 9.5 day3 兜底规则

如果当前日是 day3：

1. day3 是当前排程窗口最后一天；
2. 不允许继续往 day4 或窗口外追补；
3. 如果 activeMachines 无法完成 carryShortage + day3 日计划量，则必须在 day3 新增机台；
4. 如果没有可用机台，需要输出未排原因和未满足数量。

## 10. 每天新增机台数计算

每天新增机台数不是重新计算总机台数，而是计算增量。

公式：

```text
当天新增机台数 = 当前日满足约束后 activeMachines 数量 - 当前日前 activeMachines 数量
```

例如：

```text
day1 需要 2 台
day2 仍然需要 2 台
day3 需要 3 台
```

则输出：

```text
day1 新增 2 台
day2 新增 0 台
day3 新增 1 台
```

如果 day1 已经启用 2 台，day2 不能再重复输出新增 2 台。

## 11. 续作降模减机台判断逻辑

降模减机台只适用于续作排产场景。

续作场景下，如果同一个 SKU 当前已经在多台机台上生产，需要判断当前窗口是否仍然需要这么多机台。

判断方式：

1. 获取当前 SKU 的续作机台集合：

```text
currentRunningMachines
```

2. 按降模保留优先级对续作机台排序；
3. 从较少机台数量开始模拟，判断是否可以满足：
   - dayN 日计划；
   - 欠产追补规则；
   - 窗口总目标量；
   - 试制 / 收尾严格目标量约束；
4. 如果减少部分机台后仍然可以满足以上约束，则需要降模减机台；
5. 如果减少机台后无法满足 dayN 日计划和欠产追补约束，则不能减。

### 11.1 续作降模保留机台优先级

续作降模时，保留机台优先级调整为：

1. 胶囊已使用次数多的机台优先保留；
2. 胶囊已使用次数相同，机台编码小的机台优先保留。

被减掉的机台释放给其他 SKU 使用。

示例 1：

```text
K1105 胶囊已使用次数 = 20
K1110 胶囊已使用次数 = 12
```

降模只保留一台时，优先保留 K1105。

示例 2：

```text
K1105 胶囊已使用次数 = 12
K1110 胶囊已使用次数 = 12
```

降模只保留一台时，优先保留 K1105，因为 K1105 机台编码更小。

注意：不要再使用“胶囊次数少、机台编码大”的排序。

### 11.2 降模减机台输出结果

续作场景需要输出：

1. 原续作机台数量；
2. 当前窗口实际需要保留的最小机台数量；
3. 需要减少的机台数量；
4. 保留机台列表；
5. 被释放机台列表；
6. 降模原因；
7. 如果不能降模，输出不能降模原因。

判断：

```text
需要减少机台数 = 当前续作机台数 - 最小可行续作机台数
```

如果结果 > 0，执行降模减机台。

如果结果 <= 0，不执行降模减机台。

## 12. 同 SKU 多机台机台收尾错峰 / 尾量错开

### 12.1 规则定位

同 SKU 多机台机台收尾错峰 / 尾量错开，是针对同一个 SKU 在多台机台上生产时，多个机台在同一班次同时完成自身尾量的场景。

注意：这里是“机台收尾”，不是“SKU 收尾”。

该规则不改变 SKU 总目标量，不改变 SKU 是否收尾的判断，只调整同 SKU 多机台之间的尾量落位。

核心目的：

```text
释放一台机台资源，使该机台可以提前进入换模，间接实现换模错开。
```

### 12.2 触发条件

仅当同时满足以下条件时，才允许触发机台收尾错峰 / 尾量错开：

1. 同一个 SKU 在多台机台上生产；
2. 多台机台在同一班次同时完成自身尾量；
3. 同时收尾发生在早班或中班；
4. 存在一台尾量最小的机台，作为 releaseMachine；
5. 存在另一台同 SKU 机台，作为 receiveMachine；
6. receiveMachine 下一个班次可以承接 releaseMachine 的尾量；
7. 承接后不超过 receiveMachine 下一个班次的有效产能；
8. 调整后 SKU 总排产量不变；
9. 调整后不会影响其他 SKU 已排计划；
10. 调整后可以释放 releaseMachine，使其提前进入换模。

### 12.3 早班 / 中班才允许错峰

如果同 SKU 多机台在早班或中班同时机台收尾，可以考虑做收尾错峰 / 尾量错开。

原因：

1. 早班 / 中班后续仍然存在可换模、可开产的时间空间；
2. 释放一台机台后，可以让该机台提前进入换模；
3. 可以间接实现换模错开，减少多个机台同时换模造成的资源冲突。

### 12.4 晚班同时收尾不强制错峰

如果同 SKU 多机台收尾发生在晚班，则不需要为了错开换模而强行调整尾量分配。

原因：

1. 晚班不能换模；
2. 即使多个机台在晚班同时收尾，新 SKU 也不能在晚班立即换模；
3. 新 SKU 只能等到后续早班换模、中班开产；
4. 如果为了错开收尾而把部分尾量从晚班挪到早班或中班，反而可能浪费产能。

因此：

```text
晚班同时机台收尾，默认不触发收尾错峰 / 尾量错开。
```

### 12.5 releaseMachine 选择规则

当同 SKU 多机台在同一早班或中班同时收尾时：

1. 选择尾量最小的一台机台作为 releaseMachine；
2. 将 releaseMachine 的尾量尝试挪到另一台同 SKU 机台的下一个班次；
3. releaseMachine 原班次尾量清空后，该机台可以提前释放进入换模。

如果存在多台尾量相同的机台，可以按项目现有机台排序或机台编码稳定排序，保证结果可重复。

### 12.6 receiveMachine 下一个班次承接口径

receiveMachine 下一个班次不是“完全不能有计划”，而是“不能被其他 SKU 占用”。

最终口径如下：

1. receiveMachine 下一个班次不能被其他 SKU 占用；
2. 如果 receiveMachine 下一个班次已有当前 SKU 生产计划，且仍有剩余容量，则优先承接尾量；
3. 如果 receiveMachine 下一个班次为空，且满足当前 SKU 排产约束，也可以承接尾量；
4. 如果 receiveMachine 下一个班次已被其他 SKU 占用，禁止承接；
5. 如果 receiveMachine 下一个班次已有锁定计划且不可调整，禁止承接；
6. 承接后不能超过该班次有效产能；
7. 承接后不能改变 SKU 总排产量；
8. 承接后不能影响其他 SKU 已排计划。

### 12.7 receiveMachine 优先级

receiveMachine 优先级：

1. 下一个班次已有当前 SKU 计划，且仍有剩余容量的机台；
2. 下一个班次为空，且满足当前 SKU 排产约束的机台。

禁止再写以下口径：

```text
下一个班次未被其他 SKU 或计划占用
```

原因是该说法会与“下一个班次已有当前 SKU 计划且仍有剩余容量时优先承接”产生矛盾。

### 12.8 禁止错峰场景

以下情况禁止做机台收尾错峰 / 尾量错开：

1. 同时收尾发生在晚班；
2. receiveMachine 下一个班次被其他 SKU 占用；
3. receiveMachine 下一个班次已有锁定计划且不可调整；
4. receiveMachine 下一个班次没有剩余容量；
5. 承接后会超过 receiveMachine 下一个班次有效产能；
6. 承接后会改变 SKU 总排产量；
7. 承接后会影响其他 SKU 已排计划；
8. 调整后不能释放 releaseMachine；
9. 调整后不能带来换模错开收益；
10. 调整会导致产能浪费。

### 12.9 调整效果

调整前：

```text
同 SKU 多台机台在同一早班 / 中班同时收尾
=> 多台机台可能同时进入换模等待
```

调整后：

```text
将尾量最小机台的尾量挪到另一台同 SKU 机台的下一个班次
=> 尾量最小机台提前释放
=> 可提前进入换模
=> 间接实现换模错开
```

## 13. SKU 类型约束

### 13.1 试制 SKU

1. 只能使用单控机台；
2. 严格目标量；
3. 不允许超排；
4. 多机台也只能使用单控机台；
5. 如果单控机台不足，输出未排原因。

### 13.2 量试 / 小批量 SKU

1. 优先使用单控机台；
2. 满足条件时可使用普通机台；
3. 非收尾时，已启用机台可以按规则补满有效班次；
4. 但不能为了单纯补满空闲产能而新增机台。

### 13.3 正规 SKU

1. 优先使用普通机台；
2. 满足条件时可使用单控机台；
3. 非收尾时，已启用机台可以按规则补满有效班次；
4. 但不能为了单纯补满空闲产能而新增机台。

### 13.4 收尾 SKU

1. 收尾判断针对 SKU 整体；
2. 收尾目标量 = MAX(余量, 胎胚库存)；
3. 严格目标量；
4. 不允许超排。

## 17. 最终实现目标

### 17.1 新增排产 / 换模场景

能够根据 dayN 日计划、shortageLookAheadDays 欠产追补和最小增量原则，判断：

1. 是否需要拆分多机台；
2. 哪一天加机台；
3. 每天加几台；
4. 最终启用几台机台。

### 17.2 换活字块场景

能够先使用当前可衔接的换活字块机台进行排产，并判断：

1. 当前换活字块机台窗口有效产能；
2. 当前换活字块机台可承接数量；
3. 是否存在剩余量；
4. remainingQtyForNewSchedule 的数量；
5. 是否需要将剩余量回流到新增排产 / 换模阶段。

如果存在 remainingQtyForNewSchedule，新增排产 / 换模阶段需要继续判断：

1. 哪一天需要通过换模新增机台；
2. 每天新增几台；
3. 最终通过换模启用几台机台；
4. 是否仍存在未满足数量及原因。

换活字块具体匹配逻辑不在本文档中展开，以项目已有规则为准。

### 17.3 续作排产场景

能够根据当前续作机台数量、dayN 日计划、shortageLookAheadDays 欠产追补和最小机台数模拟，判断：

1. 是否需要降模减机台；
2. 保留哪些机台；
3. 释放哪些机台；
4. 为什么可以降模；
5. 为什么不能降模。

续作降模保留排序：

```text
胶囊已使用次数多优先；胶囊次数相同，机台编码小优先。
```

### 17.4 同 SKU 多机台机台收尾错峰 / 尾量错开

能够在同 SKU 多机台早班 / 中班同时机台收尾时，判断是否可以将尾量最小机台的尾量挪到另一台同 SKU 机台的下一个班次，从而释放机台提前进入换模。

要求：

1. 晚班同时收尾不强制错峰；
2. receiveMachine 下一个班次不能被其他 SKU 占用；
3. receiveMachine 下一个班次已有当前 SKU 计划且仍有剩余容量时优先承接；
4. receiveMachine 下一个班次为空且满足约束时也可承接；
5. 承接后不能超过有效产能；
6. 承接后不能改变 SKU 总排产量；
7. 承接后不能影响其他 SKU 已排计划。

## 18. 最终口径摘要

```text
新增排产 / 换模场景，使用 dayN 日计划 + shortageLookAheadDays 欠产追补 + 最小增量模拟，判断哪一天加几台机台。

换活字块场景，先由当前可衔接机台承接排产；如果单台机台产能无法满足窗口 dayN 日计划量和欠产追补要求，剩余量进入新增排产 / 换模阶段，通过换模分配到其他机台。

续作排产场景，不做加机台，而是用同一套产能模拟反向判断是否可以降模减机台；降模保留机台按“胶囊次数多、机台编码小”排序。

同 SKU 多机台机台收尾错峰只针对早班 / 中班有换模收益的场景；晚班不能换模，因此晚班同时收尾不强制错峰。

receiveMachine 下个班次不是完全不能有计划，而是不能被其他 SKU 占用；如果已有当前 SKU 计划且有剩余容量，反而优先承接。

历史允许追补欠产统一作为 day1 初始 carryShortage，或等价并入 day1 需求，保证窗口目标量和逐日模拟一致。
```
