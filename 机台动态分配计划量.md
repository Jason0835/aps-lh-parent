你是一名资深 Java 开发工程师，同时也是硫化排程算法专家。请基于当前项目已有代码，最小改动实现“SKU 多机台动态分配计划量”能力。

一、核心背景

当前硫化排程项目的核心模型是：

SKU 连续排 8 个班次。

也就是说：
1. 一个 SKU 一旦在某台机台开产，需要连续排产；
2. 中间不能随意断开；
3. 当前方案不能破坏这个核心约束；
4. 需要支持一个 SKU 在多台机台上生产；
5. 每台机台上的生产段必须连续。

二、本次实现目标

实现以下能力：

1. 正式/量试非收尾场景，支持多机台动态补量；
2. 正式/量试非收尾场景，非最后机台一旦开产，必须排满窗口后续所有可用班次；
3. 正式/量试非收尾场景，最后一台机台只排到满足窗口目标量，并补满最后一个已开班班次；
4. 试制非收尾场景，严格按照计划量排产，禁止为了补满班次而超排；
5. 收尾场景，严格按照 max(余量, 胎胚库存) 排产，禁止为了补满班次而超排；
6. 共用胎胚/同胎胚的换模班次必须错开；
7. 支持日计划欠产在窗口内滚动追补；
8. 保持现有主流程“SKU 连续排 8 班”的模式，尽量最小改动，不要大规模重构。

三、计划量规则

1. 正式/量试，且非收尾：

窗口最低目标量 = day1 + day2 + day3

该目标量是最低满足量，不是绝对上限。

允许少量超排，但超排只能来自：
最后一台机台的最后一个已开班班次补满。

禁止为了超排而额外开启后续班次。

2. 试制，且非收尾：

窗口目标量 = day1 + day2 + day3

该目标量是严格上限。

达到目标量后禁止继续追加。
最后一个班次即使有剩余产能，也不能补满。
不允许超排。

3. 收尾：

收尾目标量 = max(余量, 胎胚库存)

收尾场景严格按照目标量排产。
即使班次有剩余产能，也禁止补满。
不允许超排。

四、机台角色定义

需要动态判断每台机台的排产角色。

1. FULL_RUN_MACHINE：非最后机台

判断条件：

当前已排量 + 当前机台从开产班次到窗口结束的最大可排量 < 窗口目标量

说明：
当前机台即使排满窗口后续所有班次，也无法满足窗口目标量。

排产规则：
从开产班次开始，排满到窗口结束；
每个可用生产班次都排满班产；
中间不能断班；
排完后继续寻找下一台机台补量。

2. TAIL_MACHINE：最后一台机台

判断条件：

当前已排量 + 当前机台从开产班次到窗口结束的最大可排量 >= 窗口目标量

说明：
当前机台已经足以补齐窗口目标量。

排产规则：
只连续排到满足窗口目标量；
正式/量试非收尾场景，最后一个已开班班次需要补满班产；
试制/收尾场景，严格按剩余目标量截断；
达标后，后续班次不再开产。

六、共用胎胚换模错开规则

需要支持同胎胚/共用胎胚换模错开。

规则：
同一个 greenTireGroupKey 在同一个班次只能安排一次换模。

注意：
不能只按 materialCode 判断。
不同物料可能共用胎胚；
相同物料在多台机台生产也认为属于共用胎胚场景。

建议在排程上下文中维护：

Map<String, Set<String>> greenTireChangeoverShiftMap

key：greenTireGroupKey
value：该胎胚分组已经安排换模的班次集合

如果某候选机台的换模班次与同 greenTireGroupKey 已有换模班次冲突，则顺延到下一个可用班次。

例如：
K1105 C1 换模，则记录：
greenTireGroupKey -> C1

K1110 如果也想 C1 换模，则冲突，需要顺延到 C2 换模，C3 开产。

七、日计划滚动欠产规则

day1/day2/day3 不应理解为每天必须绝对完成。
由于换模、共用胎胚错开、机台可用性等约束，日计划允许在窗口内滚动追补。

建议定义：
dayN 是日计划节奏控制；
窗口总量是硬目标；
day1 欠产可以滚动到 day2；
day2 欠产可以滚动到 day3；
最终窗口累计量需要满足目标量。

当前案例：
day1 计划 96，实际 16，欠 80；
day2 计划 48，实际 96，累计仍欠 32；
day3 计划 14，实际 48，最终累计 160，满足 >=158。

九、核心算法伪代码

int targetQty = calculateTargetQty(skuPlan, policy);
int scheduledQty = 0;

List<MachineProductionSegment> segments = buildMachineProductionSegments(
        skuPlan,
        machineCandidates,
        shiftList,
        scheduleContext
);

sortMachineSegments(segments);

for (MachineProductionSegment segment : segments) {
    if (scheduledQty >= targetQty) {
        break;
    }

    int maxQty = segment.getMaxQtyToWindowEnd();

    boolean isTailMachine;

    if (policy.isFullRunForNonTailMachine()) {
        isTailMachine = scheduledQty + maxQty >= targetQty;
    } else {
        // 试制/收尾不启用非最后机台满排规则
        isTailMachine = true;
    }

    if (!isTailMachine) {
        segment.setRole(MachineScheduleRole.FULL_RUN_MACHINE);

        int qty = scheduleFullRunMachine(
                skuPlan,
                segment,
                shiftList,
                scheduleContext
        );

        scheduledQty += qty;
        continue;
    }

    segment.setRole(MachineScheduleRole.TAIL_MACHINE);

    int remainingQty = targetQty - scheduledQty;

    if (remainingQty <= 0) {
        break;
    }

    int tailPlanQty;

    if (policy.isAllowFillStartedShift()) {
        tailPlanQty = roundUpToShiftCapacity(
                remainingQty,
                segment.getShiftCapacity()
        );
    } else {
        tailPlanQty = remainingQty;
    }

    int qty = scheduleTailMachine(
            skuPlan,
            segment,
            shiftList,
            tailPlanQty,
            policy,
            scheduleContext
    );

    scheduledQty += qty;

    break;
}

validateScheduleResult(
        skuPlan,
        policy,
        targetQty,
        scheduledQty,
        scheduleContext
);

十、需要实现的关键方法

1. calculateTargetQty

规则：
如果是收尾：
targetQty = max(余量, 胎胚库存)

否则：
targetQty = day1 + day2 + day3

2. buildMachineProductionSegments

需要计算每台候选机台：
- 是否能生产当前 SKU；
- 是否需要换模；
- 最早换模班次；
- 是否存在共用胎胚换模冲突；
- 冲突则顺延；
- 换模后最早开产班次；
- 从开产班次到窗口结束的最大可排量。

3. scheduleFullRunMachine

用于非最后机台。
从开产班次开始，排满到窗口结束。
中间不能断班。
每个可用班次排满班产。

4. scheduleTailMachine

用于最后一台机台。
按 tailPlanQty 连续排产。
正式/量试非收尾允许最后班次补满。
试制/收尾严格按剩余量截断。
达到目标后停止，不再继续开启后续班次。

5. validateScheduleResult

需要校验：
- 正式/量试非收尾：实际排产量 >= 目标量；
- 正式/量试非收尾：实际排产量 - 目标量 < 最后一台机台单班产能；
- 试制/收尾：实际排产量 <= 目标量；
- 非最后机台开产后是否连续排满到窗口结束；
- 最后一台机台是否连续排产，停止后没有重新开产；
- 同 greenTireGroupKey 是否存在同班次换模冲突；
- 胎胚库存是否足够；
- 班次是否可用；
- 机台是否可用。
