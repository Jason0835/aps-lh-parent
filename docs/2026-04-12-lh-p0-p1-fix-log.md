# 硫化排程系统 P0/P1 最小风险修复记录

## A 批次：执行互斥与原子替换落库

### 风险点

- 同厂同目标日排程并发无法互斥。
- `S4.1` 提前删库会导致失败后目标日无数据或半套数据。
- `S4.6` 保存异常会被静默中断，不利于事务回滚。

### 改动摘要

- 新增 `ScheduleExecutionGuard`，在 `LhScheduleServiceImpl` 入口按 `factory + scheduleTargetDate` 获取 Redis 锁。
- `PreValidationHandler` 移除重复排程伪校验与前置删库，只保留 MES 已发布校验和批次号生成。
- 新增 `SchedulePersistenceService`，统一在事务中完成目标日旧数据删除与新批次结果插入。
- `AbsScheduleStepHandler` 新增异常直抛钩子，`ResultValidationHandler` 改为保存异常直接向上抛出。

### 验证结果

- `ExecuteScheduleConcurrencyGuardTest`
- `SchedulePersistenceRollbackTest`
- `mvn -pl aps-lh -q test`

### 回退点

- 若需要回退，仅恢复 `LhScheduleServiceImpl` 入口互斥、`PreValidationHandler` 与 `ResultValidationHandler` 的旧逻辑，并移除 `ScheduleExecutionGuard` / `SchedulePersistenceService`。

## B 批次：新增规格时间轴闭环

### 风险点

- 新增规格排产时换模、首检、开产时间链断裂。
- 同机连续新增时可能出现状态未推进与排程重叠。
- 结果缺 `specEndTime` / `tdaySpecEndTime` / `mouldCode`。

### 改动摘要

- `NewSpecProductionStrategy` 改为显式计算 `machineReadyTime -> mouldChangeStartTime -> mouldChangeEndTime -> inspectionStartTime -> productionStartTime`。
- 新增规格结果显式写入 `specEndTime`、`tdaySpecEndTime`、`mouldCode`。
- 新增规格落机后立即回写 `MachineScheduleDTO` 的当前物料、前规格、前尺寸和 `estimatedEndTime`。
- `ResultValidationHandler` 生成换模计划时按机台和时间顺序滚动推导前规格，不再直接读取最终机台快照。

### 验证结果

- `NewSpecTimelineRegressionTest`
- `MachineOccupationRegressionTest`
- `mvn -pl aps-lh -q test`

### 回退点

- 若需回退，仅恢复 `NewSpecProductionStrategy` 的旧时间计算和 `ResultValidationHandler` 的旧换模计划生成逻辑。

## C 批次：机台状态初始化补齐

### 风险点

- 机台前规格、寸口、清洗、保养、维修和基线结束时间字段未完整初始化，导致后续排序和产能限制失效。

### 改动摘要

- `DataInitHandler` 补齐在机物料描述、前规格、前尺寸。
- 从清洗计划、保养计划、停机计划中灌入清洗/保养/维修状态。
- 初始化 `estimatedEndTime`：优先前日最后 `specEndTime`，其次机台基线，再退化到窗口首班开始。
- 新增 `initialMachineScheduleMap` 快照供换模计划和回归校验使用。
- `LhScheduleTimeUtil` 新增保养时间多格式解析 helper。

### 验证结果

- `MachineStateInitRegressionTest`
- `DefaultCapacityCalculateStrategyRegressionTest`
- `mvn -pl aps-lh -q test`

### 回退点

- 若需回退，仅恢复 `DataInitHandler` 的旧字段初始化逻辑并移除 `initialMachineScheduleMap` 的使用。

## D 批次：欠产传导、收尾/优先级、结果强校验

### 风险点

- 欠产/超产通过修改昨天计划实现，无法稳定传导到当天待排量。
- 收尾天数存在偏移。
- `delayDays`、`embryoStock` 的未知值会被当成 `0` 参与排序或库存削减。
- 关键字段缺失时只记日志，不阻断坏数据落库。

### 改动摘要

- `ScheduleAdjustHandler` 改为按物料汇总前日净欠产/超产，并回写当天 `pendingQty` / `surplusQty`。
- `DefaultEndingJudgmentStrategy` 改为按 3 班制向上取整折算天数。
- `DefaultSkuPriorityStrategy` 仅让已知 `delayDays` 参与排序。
- `ContinuousProductionStrategy` 与 `NewSpecProductionStrategy` 统一按 `embryoStock >= 0` 视为可信库存。
- `ResultValidationHandler` 对 `batchNo`、`factoryCode`、`lhMachineCode`、`materialCode`、`scheduleType`、`specEndTime`、`mouldCode` 升级为强校验，校验失败直接阻断持久化。

### 验证结果

- `ScheduleAdjustCarryForwardRegressionTest`
- `EndingDaysRegressionTest`
- `ResultValidationBlockingTest`
- `mvn -pl aps-lh -q test`

### 回退点

- 若需回退，仅恢复 `ScheduleAdjustHandler`、`DefaultEndingJudgmentStrategy`、`DefaultSkuPriorityStrategy`、`ResultValidationHandler` 的旧逻辑。
