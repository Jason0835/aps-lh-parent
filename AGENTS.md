# 总体工作原则

2. 优先基于现有代码结构改造，禁止大范围重构。
3. 优先复用项目已有类、方法、枚举、常量、工具类、Mapper、Service。
4. 修改前必须先分析影响范围，包括：
   - 入口方法
   - 核心排程逻辑
   - 数据初始化逻辑
   - 月计划 / 日计划扣减逻辑
   - 续作逻辑

# Java 基础编码规范

1. JDK 版本使用 Java 8，语法遵循 Java 8+ 规范。
2. 缩进使用 4 个空格，禁止使用 Tab。
3. 类名使用大驼峰：`UpperCamelCase`。
4. 方法名、变量名使用小驼峰：`lowerCamelCase`。
5. 常量使用大写加下划线：`MAX_RETRY_COUNT`。
6. 包名全部小写：`com.xxx.xxx.module.service`。
7. 方法行数尽量不超过 80 行。
8. 类文件尽量不超过 1000 行。
9. 禁止魔法值，必须抽取为常量或枚举。
10. 集合初始化时必须指定合理初始容量，避免频繁扩容。
11. 可以使用 Lombok 的 `@Data`。
12. 所有实体类必须实现 `Serializable`。
13. 不使用内部类，除非只能通过内部类实现。
14. 字符串循环拼接必须使用 `StringBuilder`，禁止在循环中使用 `+` 拼接。
15. 遵循阿里 Java 开发规范。

# 判空与工具类规范

1. 字符串判空必须使用：

   ```java
   StringUtils.isEmpty(str)
````

2. 字符串非空判断必须使用：

   ```java
   StringUtils.isNotEmpty(str)
   ```

3. 禁止使用以下原生字符串判空方式：

   ```java
   str == null
   "".equals(str)
   str.length() == 0
   ```

4. 集合判空必须使用：

   ```java
   CollectionUtils.isEmpty(collection)
   ```

5. 对象判空优先使用：

   ```java
   Objects.isNull(obj)
   Objects.nonNull(obj)
   ```

6. 数字、`BigDecimal` 判空与比较必须使用项目已有工具类；如果项目没有统一工具类，再使用标准写法。

7. `BigDecimal` 比较禁止直接使用 `equals()` 判断数值大小。

8. 日期工具统一使用 Hutool 或 `java.time` 包。

9. 禁止使用 `SimpleDateFormat`。

10. 优先使用Hutool已有的工具类

# SQL 与 MyBatis-Plus 规范

## 单表新增 / 修改

1. 普通新增、修改优先使用：

   ```java
   mapper.insert(entity)
   mapper.updateById(entity)
   ```

2. 简单固定 SQL 增改可以使用注解 SQL。

3. 批量插入、动态字段更新、复杂 SQL 使用 XML。

4. 禁止单表简单新增 / 修改写 XML。

## 单表查询 / 删除

1. 单表查询优先使用 MyBatis-Plus 原生 `LambdaQueryWrapper`。
2. 单表删除优先使用 MyBatis-Plus 原生能力。
3. 单表查询 / 删除原则上不写 XML。

## 多表 / 复杂查询

1. 多表关联、复杂聚合、动态查询可以使用 XML。
2. XML 中 SQL 要保持可读性。
3. SQL 字段必须明确，不要滥用 `select *`。
4. 查询条件要贴合业务，不允许增加无业务依据的兜底条件。
