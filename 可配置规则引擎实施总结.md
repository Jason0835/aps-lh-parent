```java
@Override
public int getDryIceIntervalDays(String factoryCode) {
    return getIntValue(factoryCode, "DRY_ICE_INTERVAL_DAYS", 
            LhScheduleConstant.DRY_ICE_INTERVAL_DAYS);
}

@Override
public double getCapsulePreheatHours(String factoryCode) {
    String value = getParamValue(factoryCode, "CAPSULE_PREHEAT_HOURS");
    if (value == null || value.trim().isEmpty()) {
        return LhScheduleConstant.CAPSULE_PREHEAT_HOURS.doubleValue();
    }
    try {
        return Double.parseDouble(value.trim());
    } catch (NumberFormatException e) {
        log.warn("参数解析失败，factoryCode={}, paramCode={}, value={}, 使用默认值：{}", 
                factoryCode, "CAPSULE_PREHEAT_HOURS", value, 
                LhScheduleConstant.CAPSULE_PREHEAT_HOURS);
        return LhScheduleConstant.CAPSULE_PREHEAT_HOURS.doubleValue();
    }
}
```

## 🔧 技术亮点

### 1. 反射调用机制

```java
private static int getRuleValue(String factoryCode, String paramCode, int defaultValue) {
    IScheduleRuleEngine ruleEngine = getRuleEngine();
    if (ruleEngine == null || factoryCode == null) {
        return defaultValue;
    }
    try {
        java.lang.reflect.Method method = IScheduleRuleEngine.class.getMethod(paramCode, String.class);
        Object result = method.invoke(ruleEngine, factoryCode);
        return result instanceof Integer ? (Integer) result : defaultValue;
    } catch (Exception e) {
        log.debug("规则引擎获取参数失败，使用默认值：paramCode={}, factoryCode={}", paramCode, factoryCode);
        return defaultValue;
    }
}
```

### 2. 缓存刷新机制

```java
private void refreshCache(String factoryCode) {
    try {
        // 从数据库加载参数列表
        List<LhParams> paramsList = paramsMapper.selectList(
                new LambdaQueryWrapper<LhParams>()
                        .eq(LhParams::getFactoryCode, factoryCode)
                        .eq(LhParams::getIsDelete, DeleteFlagEnum.NORMAL.getCode()));
        
        // 转换为 Map
        Map<String, String> params = new HashMap<>();
        if (paramsList != null) {
            for (LhParams param : paramsList) {
                if (param.getParamCode() != null && param.getParamValue() != null) {
                    params.put(param.getParamCode(), param.getParamValue());
                }
            }
        }
        
        paramsCache.put(factoryCode, params);
        cacheTimestamp.put(factoryCode, System.currentTimeMillis());
        log.debug("刷新参数缓存，factoryCode={}, 参数数量：{}", factoryCode, params.size());
    } catch (Exception e) {
        log.error("刷新参数缓存失败，factoryCode={}", factoryCode, e);
    }
}
```

## ⚠️ 注意事项

### 1. 数据库兼容性

确保 `T_LH_PARAMS` 表包含以下字段:
- `FACTORY_CODE` (VARCHAR) - 工厂编码
- `PARAM_CODE` (VARCHAR) - 参数代码
- `PARAM_VALUE` (VARCHAR) - 参数值
- `PARAM_NAME` (VARCHAR) - 参数名称
- `REMARK` (VARCHAR) - 备注
- `IS_DELETE` (INT) - 逻辑删除标志

### 2. Spring 上下文初始化

规则引擎需要 Spring 容器管理，确保:
- 类上有 `@Component` 或 `@Service` 注解
- 包扫描路径包含 `com.zlt.aps.lh.engine.rule`

### 3. 缓存一致性

- 规则修改后最多 5 分钟生效
- 紧急情况下可手动清除缓存：`ruleEngine.clearCache("F001")`

### 4. 参数格式验证

建议在应用层增加参数格式验证:
```java
// 正则表达式验证数字
if (!paramValue.matches("^\\d+(\\.\\d+)?$")) {
    throw new IllegalArgumentException("参数值格式错误");
}
```
